Click Confirm

Convenient and elegant inline user confirmation of UI events.

View on GitHub

This VueJS component prompts a user to confirm a click action as deliberate before allowing the event to proceed. The component wraps around any nested clickable element and intercepts any click event with a prompt request. The prompt appears immediately adjacent to the button without otherwise interrupting the page, making the experience faster and more comfortable for the user than a typical Javascript prompt.

Getting Started

Click Confirm depends on the following in your production environment:
Dependency Version Tested with Notes
VueJS 2.0+ 2.1.10 Why else would you be here?
Bootstrap 4.0.0 (alpha) 4.0.0-alpha.6 Uses CSS only, Bootstrap's Javascript package is not required.
Font Awesome 4.0+ 4.7 This may be safely excluded in order to not show icons on the buttons, or may be replaced with another icon typeset. See below.
Click Confirm depends on Bootstrap Vue for use of its Popover component. Don't worry, this component will be pulled in automatically if you are not already using it.
Pull Click Confirm in to your project via npm: 
npm install click-confirm --save
The recommended use method is to include the ClickConfirm.vue file as part of your package generation. 
import Vue from 'vue'
import ClickConfirm from 'click-confirm/src/ClickConfirm.vue'

Vue.component('clickConfirm', ClickConfirm);
If, for some reason, this is not an available option you can include the pre-bundled script, however this is inefficient and not recommended for a production environment. 
import Vue from 'vue'
import ClickConfirm from 'click-confirm'

Vue.component('clickConfirm', ClickConfirm);

The Basics

Click Confirm will flexibly intercept any click events. This includes both native DOM events and VueJS events. Any emitted 'click' events will be intercepted and re-transmitted if confirmed via the popup prompt.

Typical URL link

URL link as button 

<click-confirm>
  <a href="http://www.example.com" target="_blank">Typical URL link</a>
</click-confirm>

<click-confirm>
  <a href="http://www.example.com" class="btn btn-primary"
          role="button" target="_blank"
  >URL link as button</a>
</click-confirm>

<click-confirm>
  <button class="btn btn-primary"
          onclick="alert('It worked!')"
  >DOM 'onclick' event</button>
</click-confirm>

<click-confirm>
  <button class="btn btn-primary"
          @click="successAlert"
  >VueJS @click event</button>
</click-confirm>
Subsequent demonstrations all use a Javascript alert for brevity.

Properties

All properties on the Click Confirm component are optional. Demonstrations are provided below.
Property Data Type(s) Default Description
button-size "sm", "" (medium), or "lg" "" Render size of confirmation buttons, per Bootstrap 4.
copy-attributes String, Array ["href", "target"] Attributes from enclosed element which will be copied to the 'yes' button. Intended for copy a URL to the 'yes' button, but it is customizable.
disabled Boolean false Suppresses the click-confirm dialogue and allows the click event to proceed as normal.
messages Object, formatted per Default { title: 'Are you sure?', yes: 'Yes', no: 'No' } Messages displayed within the confirmation dialogue. All or part of the object properties may be included; any missing will be displayed per the default values. This design is intended to make multi-language support straightforward.
no-class String, Array, Object "btn btn-secondary" Class to use for the 'No' button, can be used for further styling.
no-icon String, Array, Object "fa fa-times" Typeset icon visible within the 'No' button.
placement "top", "bottom", "left", or "right" "top" Display location of the confirmation popover.
yes-icon String, Array, Object "fa fa-check" Typeset icon visible within the 'Yes' button.
yes-class String, Array, Object "btn btn-primary" Class to use for the 'Yes' button, can be used for further styling.

Placement

Like Bootstrap Popover, the confirmation prompt can be displayed on any side of the target. 

<click-confirm placement="bottom">
  <button class="btn btn-primary" @click="successAlert">Bottom</button>
</click-confirm>

<click-confirm placement="right">
  <button class="btn btn-primary" @click="successAlert">Right</button>
</click-confirm>

<click-confirm placement="left">
  <button class="btn btn-primary" @click="successAlert">Left</button>
</click-confirm>

<click-confirm>
  <button class="btn btn-primary" @click="successAlert">Top (default)</button>
</click-confirm>

Multi-Language Support

Supporting multiple languages is simple. The object-based structure is intended to make implementing multiple languages straightforward. 

<click-confirm>
  <button class="btn btn-primary" @click="successAlert">English (default)</button>
</click-confirm>

<click-confirm
          :messages="{title: 'Êtes-vous sûr?', yes: 'Oui', no: 'Non'}">
  <button class="btn btn-primary" @click="successAlert">Français</button>
</click-confirm>

<click-confirm
          :messages="{title: 'I am Groot?', yes: 'I am Groot', no: 'I am Groot'}">
  <button class="btn btn-primary" @click="successAlert">Groot</button>
</click-confirm>

Button Icons

Click Confirm ships configured to use Font Awesome's 'check' (for yes) and 'times' (for no) icons. You can provide alternate CSS classes to apply to the button's 'span' element. Any other CSS-based icon library, such as Glyphicons, may also be used. In order to allow this, you must include all prefix class names required by your desired icon library. 

<click-confirm>
  <button class="btn btn-primary" @click="successAlert">Check and X (default)</button>
</click-confirm>

<click-confirm yes-icon="fa fa-thumbs-up" no-icon="fa fa-thumbs-down">
  <button class="btn btn-primary" @click="successAlert">All thumbs</button>
</click-confirm>

<click-confirm yes-icon="fa fa-smile-o fa-spin" no-icon="fa fa-frown-o fa-spin">
  <button class="btn btn-primary" @click="successAlert">Turn that frown upside-down</button>
</click-confirm>

Button Classes

Click Confirm ships configured to use Bootstraps's 'btn-primary' (for yes) and 'btn-secondary' (for no) classes. You can provide alternate CSS classes to apply to the button's 'a' element, thus allowing for more detailed styling of the buttons. The styles provides will be merged with the 'button-size' classes, so the button size should not be included in this property. 

<click-confirm>
  <button class="btn btn-primary" @click="successAlert">Check and X (default)</button>
</click-confirm>

<click-confirm yes-class="btn btn-success" no-class="btn btn-danger">
  <button class="btn btn-primary" @click="successAlert">All thumbs</button>
</click-confirm>

Button Size

The confirmation button size is customizable. 

<click-confirm>
  <button class="btn btn-primary" @click="successAlert">Medium (default)</button>
</click-confirm>

<click-confirm button-size="sm">
  <button class="btn btn-primary" @click="successAlert">Small</button>
</click-confirm>

<click-confirm button-size="lg">
  <button class="btn btn-primary" @click="successAlert">Large</button>
</click-confirm>

Disable

The confirmation feature can be programmatically suppressed, which will allow the click event to proceed uninterrupted. 

<click-confirm>
  <button class="btn btn-primary" @click="successAlert">Enabled (default)</button>
</click-confirm>

<click-confirm :disabled="true">
  <button class="btn btn-primary" @click="successAlert">Small</button>
</click-confirm>

Custom Appearance

The Click Confirm popover is enclosed in a CSS element and class "div.click-confirm". This allows you to customize the appearance using CSS scoped within this parent class. Please study the code on GitHub or a rendered example from above to learn how to target the desired elements.
